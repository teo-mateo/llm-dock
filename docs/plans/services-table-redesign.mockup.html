What changed, and why

Each decision maps to a concrete problem in the current table. All of it is implementable in ServicesTable.jsx without backend changes — the data is already there.

Was: 13 red rows

Red only means failed

Exit 0 and 137 (docker stop / SIGKILL) are normal outcomes of stopping a model — they render as neutral gray "Stopped". Red is reserved for real crashes (exit 1, 255). Today vllm-bge-m3's genuine failure drowns among eleven false alarms.

Was: alphabetical order

State groups, actionable first

Running pins to the top (it's what's using your VRAM), Failed comes second (it's what needs you), then Stopped and Not created. Within a group, alphabetical. No more scanning 19 rows to find the one that's alive.

Was: key-d45f8969… × 19

API key shown once

Every service shares the default key, so repeating it per row is pure noise and doubles row height. It moves to a single copyable chip in the toolbar, next to "Rotate key" which acts on it. A per-row key would only reappear if a service ever diverges from the default.

Was: dot + text + row tint

Status encoded once, as a pill

Three redundant encodings collapse into one pill. The row keeps just a 3px left stripe for running/failed so state is scannable even when the status column is out of your foveal view.

Was: bare "80.76 GB" text

Size in VRAM context

Right-aligned tabular numerals with a tiny bar scaled to total VRAM (97.9 GB). You see instantly that mistral-q4 nearly fills the card and qwen3-5-397b (red bar) can never fit resident. This is the column that answers "can I start this right now?" — the GPU strip above gives the free-memory anchor.

Was: ▶ ✎ ⧉ ◎ + FontAwesome

One icon language, destructive tucked away

A single consistent SVG set replaces the mixed unicode glyphs. Start/stop stays primary; Delete moves into a "⋯" overflow menu (with Rename and YAML) so a 24 px slip can't nuke a service. Secondary icons stay dim until row hover.

Was: cryptic ◎ per row

Public port as a label, not a button

Port 3301 gets a small PUBLIC tag; "make this the public port" moves into the overflow menu. An every-row icon button for a rare action was spending prime pixels on noise.

Was: "Loading GPU info…"

GPU strip is part of the frame

A slim, always-present strip above the toolbar: card name, a usage bar, used/free figures. It loads into a fixed-height slot so nothing jumps, and it gives the size bars their reference point. Count chips (2 running · 2 failed · 15 idle) summarize the table before you read it.
