# LLM-Dock - Developer Guide

## Project Overview

LLM-Dock is a Docker Compose-based dashboard for managing local LLM inference services. It supports llama.cpp (GGUF models) and vLLM with automatic GPU detection, model discovery, and Open WebUI integration.

**Access points:**
- Dashboard: `http://localhost:3399`
- React frontend (v2): `http://localhost:3399/v2`
- Open WebUI: `http://localhost:3300`

## Tech Stack

## Setup & Running

### Prerequisites
- Linux (Ubuntu 22.04)
- Docker with Compose v2 (`docker compose`)
- Python 3.10+
- NVIDIA GPU + CUDA drivers + nvidia-container-toolkit
- User must be in `docker` group

### Quick Start
```bash
git clone https://github.com/teo-mateo/llm-dock.git && cd llm-dock
./setup.sh                    # Creates venv, installs deps, generates password
./build-llamacpp.sh           # Build llama.cpp image (select GPU arch)
cd dashboard
source venv/bin/activate
python app.py                 # Start Flask server on port 3399
```

### React Frontend (v2) Dev Mode
```bash
cd dashboard/frontend
npm install
npm run dev                   # Starts Vite dev server, proxies /api to Flask :5000
```

### Build & Lint
```bash
# Backend linting
ruff check dashboard/         # if ruff is installed

# Frontend linting
cd dashboard/frontend && npm run lint

# Frontend build
cd dashboard/frontend && npm run build

# Tests
cd dashboard && pytest tests/
```

### Model Discovery Paths
- `~/.cache/huggingface/hub/` - HuggingFace cache
- `~/.cache/models/` - Generic GGUF directory
- Custom paths can be added in `model_discovery.py`

## Architecture

### Application Factory Pattern
`app.py` uses Flask's application factory pattern (`create_app`). Blueprints are registered in order:
1. `system_bp` - System info
2. `gpu_bp` - GPU statistics
3. `services_bp` - Service CRUD
4. `openwebui_bp` - Open WebUI integration
5. `benchmarks_bp` - Benchmarking
6. `chat_bp` - Chat functionality

### Authentication
All API routes use the `@require_auth` decorator from `auth.py`. Clients must send:
```
Authorization: Bearer <DASHBOARD_TOKEN>
```
Token comparison uses `secrets.compare_digest()` for constant-time safety.

### Docker Compose Management (`compose_manager.py`)
- Services are defined in `services.json` (the source of truth)
- `docker-compose.yml` is auto-generated from the template + services.json
- Uses markers `# <<<<<<< BEGIN DYNAMIC` / `# >>>>>>> END DYNAMIC` to delimit the auto-generated section
- Updates are atomic: writes to temp file, validates with `docker compose config`, then replaces
- Includes backup/rollback on failure

### Service Templates
Services are generated from Jinja2 templates in `templates/`:
- `llamacpp.j2` - llama.cpp containers (GGUF models)
- `vllm.j2` - vLLM containers (safetensors models)

### Open WebUI Integration
Services are auto-registered as OpenAI-compatible endpoints in Open WebUI when created. Each service gets a unique API key stored in `services.json`.

## Supported Engines

### llama.cpp
- **Format:** GGUF files
- **Multimodal:** Supported (mmproj files)
- **Image:** Custom build (`llm-dock-llamacpp`)
- **Benchmarking:** Built-in `llama-bench` support
- **Params:** CLI flags directly (e.g. `-ngl 99`, `-fa 1`, `-c 8192`)

### vLLM
- **Format:** safetensors
- **Image:** `vllm/vllm-openai:v0.11.0`
- **Params:** `--max-model-len`, `--gpu-memory-utilization`, `--tensor-parallel-size`, etc.

## API Routes

All routes are under the Flask app and require Bearer token auth (except static file serving).

| Blueprint | Key Endpoints | Purpose |
|-----------|--------------|---------|
| `system_bp` | `/api/system/info` | System hardware info |
| `gpu_bp` | `/api/gpu/stats` | Real-time GPU stats via nvidia-smi |
| `services_bp` | `/api/services`, `/api/services/<name>`, `/api/models` | Service CRUD, model discovery |
| `openwebui_bp` | `/api/openwebui/*` | Open WebUI integration |
| `benchmarks_bp` | `/api/benchmarks`, `/api/benchmarks/run` | Benchmark execution & results |
| `chat_bp` | `/api/chat/*` | Chat conversations & messages |

## Coding Conventions

- **Backend:** Python with type hints, logging via standard `logging` module
- **Frontend (v2):** React functional components with hooks, Tailwind CSS for styling
- **Routes:** Organized as Flask blueprints in `routes/`
- **File updates:** Always use atomic writes (temp file + rename) with backup/rollback
- **Security:** Constant-time token comparison, security headers on all responses (X-Content-Type-Options, X-Frame-Options, etc.)
- **No comments** unless explicitly requested
